## 2024-03-24 - Cloudflare D1 db.batch() vs Window Functions
**Learning:** While combining parallel `Promise.all` reads into a single query using `COUNT(*) OVER()` window functions reduces network round-trips in Cloudflare D1, it introduces a severe performance regression when paired with `LIMIT`/`OFFSET` for pagination. Window functions force the SQLite engine to evaluate joins and materialize the *entire* matching dataset before applying the limit, destroying index-based short-circuiting.
**Action:** Always use `c.env.DB.batch()` to consolidate independent parallel read queries in Cloudflare D1. It reduces HTTP network overhead without fundamentally altering the optimized query execution plans of individual statements. Ensure you correctly handle the returned multidimensional array.
