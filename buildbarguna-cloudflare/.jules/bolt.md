## 2025-02-21 - Optimize O(N) database queries in D1
**Learning:** In Cloudflare D1 (SQLite over HTTP), running N concurrent queries inside a `Promise.all` loop (such as fetching aggregate revenue per project or performing inserts) creates a significant performance bottleneck due to per-query HTTP network overhead.
**Action:** Replace sequential queries inside a loop with a single query using `GROUP BY` and an `IN` clause. Map the results back to the source objects using a `Map` to maintain O(1) lookup efficiency and handle items with zero/null values correctly. Also, replace `Promise.all(queries.map(q => q.run()))` with `c.env.DB.batch(queries)` for inserts. Ensure proper empty checks are in place before building `IN` clauses or executing batched statements to prevent errors.
