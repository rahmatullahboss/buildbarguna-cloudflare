## 2024-05-18 - Replace Promise.all with db.batch for D1 parallel reads
**Learning:** In Cloudflare D1, executing independent parallel database read queries with `Promise.all()` introduces substantial per-query HTTP network overhead. The `/portfolio` route was creating N parallel queries for user project monthly histories. `db.batch()` consolidates these into a single database network request, drastically lowering latency. However, when using `db.batch()`, you cannot chain `.all()`, `.first()`, or `.run()` to the prepared statements inside the batch array.
**Action:** Use `c.env.DB.batch([stmts])` instead of `Promise.all()` when mapping over independent read queries, and wrap the batch call in an array length check to prevent empty batch errors. Process the batch results via the `results` array of the returned objects.
