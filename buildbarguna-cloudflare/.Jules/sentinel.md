## 2025-05-08 - XSS Vulnerability in PDF Generation via HTML Interpolation
**Vulnerability:** XSS/SSRF via malicious user input processed by external PDF renderers (PDFShift, DocRaptor, Adobe). Registration forms allowed potentially malicious HTML tags (e.g., `<script>`) to be inserted into strings via unescaped string interpolation (e.g., `${reg.name_english}`) inside the HTML templates before sending them to external PDF conversion APIs.
**Learning:** External PDF generation services run headless browsers to convert HTML to PDF. If untrusted user input is directly concatenated into the HTML string, these headless browsers will execute any injected scripts or HTML. This can lead to local file reads, Server-Side Request Forgery (SSRF) against internal network services of the PDF rendering provider, or theft of API credentials if exposed to the render environment.
**Prevention:** Always escape user input before interpolating it into HTML strings, even if those strings are intended for server-side processing like PDF generation. Implement a robust `escapeHTML` helper and use it strictly for all dynamic data fields injected into HTML templates.
