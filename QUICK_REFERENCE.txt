🔴 BLOCKING ISSUES (Must Fix Before Launch)

1. CRON CPU TIMEOUT
   Location: src/cron/earnings.ts line 53-56
   Problem: Sequential batch processing times out at >50k users
   Fix: Use Promise.all() for parallel batching
   Time: 1-2 hours
   Impact: Earnings distribution fails if cron times out

2. SHARE APPROVAL RACE CONDITION
   Location: src/routes/admin.ts line 148-195
   Problem: Concurrent admins can approve same request twice
   Fix: Add share_approvals lock table with unique constraint
   Time: 2-3 hours
   Impact: Users can acquire shares beyond project limit

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🟡 IMPORTANT ISSUES (Should Fix Before 10k Users)

3. KV EVENTUAL CONSISTENCY
   Location: src/routes/auth.ts, src/middleware/auth.ts
   Problem: Logout takes 60 seconds to propagate across regions
   Fix: Migrate token blacklist from KV to D1
   Time: 2-3 hours
   Impact: Users can use stale tokens for 60 seconds after logout

4. MISSING DATABASE INDEXES
   Location: src/db/schema.sql end of file
   Problem: 4 critical indexes missing cause full table scans
   Fix: Add 4 CREATE INDEX statements
   Time: 30 minutes
   Impact: Queries 50x slower than optimal (50ms → 1ms)

5. PROJECTS N+1 SUBQUERIES
   Location: src/routes/projects.ts line 10-46
   Problem: Each project spawns a subquery (21 queries for 20 projects)
   Fix: Use LEFT JOIN + GROUP BY instead of subqueries
   Time: 30 minutes
   Impact: Project listing 10x slower (100ms → 10ms)
