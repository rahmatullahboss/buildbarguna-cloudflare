## 2025-04-26 - Optimized O(N) database operations in Cloudflare D1
**Learning:** The primary performance bottleneck in Cloudflare D1 is the network roundtrip latency for each query. In `buildbarguna-cloudflare/src/routes/company-expenses.ts`, `Promise.all` was used sequentially to perform database reads and inserts inside loops, causing N+1 latency issues. Using `GROUP BY` with an `IN (...)` clause mitigates the O(N) read overhead. D1's `db.batch()` consolidates the remaining insert loops into a single network roundtrip, significantly reducing overhead for Cloudflare Workers. Attempting to use `db.batch` on an empty array will trigger a `D1_BATCH_MUTATION_ERROR`, so an `if (arr.length > 0)` check is critical.
**Action:** Always replace `Promise.all` database queries with a single query using `GROUP BY` and `IN` for reads, mapping the results back to the source objects using a `Map` for O(1) efficiency. Use `db.batch()` for any remaining independent write operations, wrapped in a length check.
