## 2024-05-24 - Semantic Form Accessibility and Custom Toggles
**Learning:** Custom UI toggles (like the payment/withdrawal method selectors built with `<button>`) must use `role="radiogroup"` and `role="radio"` with `aria-checked` to be accessible to screen readers, instead of relying purely on visual styling. Furthermore, all form `<label>` elements must programmatically link to their `<input>` using `htmlFor` and `id` to increase click targets and ensure screen readers announce input names correctly.
**Action:** When building custom radio button-like toggles or generic form inputs, always implement the correct `aria` roles and `htmlFor`/`id` associations. Add Tailwind's `focus-visible` ring utilities to ensure keyboard navigators can see their active focus state.
